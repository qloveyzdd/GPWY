### Requirement Baseline Drift

The current interval-high rule is:

1. Start from the latest trading day.
2. Move backward while the previous day's high is strictly greater than the current candidate high.
3. Stop when the previous day's high is not greater.
4. The current candidate is the interval high.

This is the user-confirmed behavior and is covered by current unit tests. It does not satisfy the original strict local-high plus 60-day fallback wording in SCRN-04 and SCRN-05. The milestone should either update those requirements as approved adjustments or restore the original algorithm.

### REFR-05

REFR-05 is implemented:

- `stock_basics` caches listed-stock metadata.
- `daily_bars` caches latest-basis adjusted OHLCV.
- `screening_runs` and `screening_results` cache derived screening output.
- `chip_peak_runs`, `chip_peak_results`, and `chip_peak_levels` cache chip output.
- `readLatestResultsSnapshot()` reads the latest matching persisted screening/chip snapshot.

Its Partial status is a planning/verification gap: REQUIREMENTS.md is unchecked and Phase 2 has no phase verification document.
