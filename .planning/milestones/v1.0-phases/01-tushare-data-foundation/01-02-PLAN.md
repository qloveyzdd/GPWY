<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Protected browser action -> validation API | Authenticated user triggers quota-consuming external call. |
| Validation API -> Tushare Pro API | Server sends token to external service. |
| Tushare response -> SQLite/UI | Untrusted external response becomes stored/displayed summary. |
| Status API -> browser | Stored validation snapshot leaves server boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Information Disclosure | `src/lib/tushare/client.ts` | mitigate | Keep token in request body only; do not include it in return values, errors, or logs. |
| T-01-06 | Tampering | `src/lib/validation/run-basic-validation.ts` | mitigate | Validate Tushare response shapes with zod before saving summaries. |
| T-01-07 | Denial of Service | `src/app/api/validation/run/route.ts` | mitigate | Require session and keep validation explicit; page load reads SQLite only. |
| T-01-08 | Information Disclosure | `src/app/api/status/route.ts` | mitigate | Return only sanitized snapshot fields defined in UI-SPEC. |

</threat_model>
