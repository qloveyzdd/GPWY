### Interval-High Rule

The canonical rule is now consistent across project, requirements, roadmap, implementation, tests and Phase 3 verification:

1. Use the latest 60 trading-day window.
2. Start with the latest trading day as the candidate.
3. Move backward while the previous trading day's high is strictly greater than the candidate high.
4. Stop when the previous high is less than or equal to the candidate high.
5. Use the current candidate as the interval high; a latest-day new high therefore becomes the new interval high.

Regression tests cover the `002930` latest-day high, the `301608` previous-day high, equal-high stopping and the 85% boundary.

### Cache Requirement

`REFR-05` is fully implemented and verified:

- `stock_basics` and `daily_bars` persist the latest successful refresh snapshot.
- `screening_runs` and `screening_results` persist derived screening output.
- `chip_peak_runs`, `chip_peak_results` and `chip_peak_levels` persist official chip enrichment.
- Failed refreshes do not replace the latest successful cache.
- The page reads the latest persisted screening/chip snapshot.
