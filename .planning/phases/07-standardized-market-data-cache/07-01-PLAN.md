## Phase Goal

**作为**维护者，**我希望**标准化市场数据拥有稳定的自然键和原子激活边界，**从而**任何读取者都不会看到半成品缓存。

<objective>
建立 generation 化的标准化 SQLite 存储层，独立保存股票主数据、原始日线和复权因子，并提供可验证、可回滚的激活生命周期。

Purpose: 为后续动态复权、首次引导和增量刷新提供唯一持久化事实源。
Output: 标准化类型、schema、UPSERT API、日期清单、generation 生命周期与集成测试。
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 应用领域对象 -> SQLite | 未验证或重复数据进入持久化层。 |
| building generation -> active generation | 半成品数据尝试成为权威读取源。 |
| 失败清理 -> 已激活历史 | 清理范围错误可能删除可用缓存。 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02 | Tampering | `market-data-store.ts` 激活事务 | mitigate | store 内重新校验恰好 60 个 paired-success 日期后才切换 active 指针。 |
| T-07-03 | Denial of Service | SQLite 写事务 | mitigate | 网络请求全部在事务外；批量写入和激活使用短事务。 |
| T-07-04 | Tampering | failed generation 清理 | mitigate | 仅按明确的 building generation id 删除其子表记录，测试证明 active/retired 数据不受影响。 |

</threat_model>
