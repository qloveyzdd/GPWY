<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SQLite 状态 -> server DTO | 内部 generation、路径或错误细节可能被带到客户端。 |
| server DTO -> React UI | 错误来源判断可能错误标记或隐藏旧结果。 |
| 轮询状态 -> 用户交互 | 频繁更新可能破坏表格操作或重复播报。 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | refresh/results DTO | mitigate | 只暴露 mode、cacheSource 和现有脱敏摘要；测试排除 token、路径、表名和 payload。 |
| T-07-02 | Tampering | cacheSource 判定 | mitigate | 依据持久化 screening provenance 判断 legacy/normalized，不依据瞬时 UI 状态猜测。 |
| T-07-08 | Denial of Service | `StatusWorkspace` polling | accept/mitigate | 复用现有单轮询和 router refresh，不新增轮询循环或逐次 live-region 播报。 |

</threat_model>
