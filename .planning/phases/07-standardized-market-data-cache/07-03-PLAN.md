<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Tushare 响应 -> 标准化类型 | 外部字段、空数据和异常值进入应用。 |
| 长耗时 provider 调用 -> SQLite | 网络等待可能阻塞唯一写者。 |
| building generation -> active generation | 部分日期或单侧请求成功可能被误判完整。 |
| 服务器错误 -> refresh 状态/UI | token、路径或原始 payload 可能泄露。 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | provider/refresh error handling | mitigate | 复用错误分类和脱敏，持久化前移除 token、headers、路径和原始 payload。 |
| T-07-02 | Tampering | bootstrap validation | mitigate | 以 60 条 manifest 的 daily/factor 双成功为唯一激活条件，并由 store 二次验证。 |
| T-07-03 | Denial of Service | bootstrap orchestration | mitigate | provider 调用全部在事务外且保持串行；失败 generation 立即清理，避免重复失败累积。 |
| T-07-07 | Repudiation | refresh job mode | mitigate | 记录安全的 bootstrap mode 和完成/失败状态，不记录秘密或内部 generation 细节到 UI。 |

</threat_model>
