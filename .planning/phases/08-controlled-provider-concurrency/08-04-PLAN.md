<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Workflow fan-out -> scheduler | Promise.all 可瞬间创建大量任务，但不能绕过真实启动上限。 |
| Provider completion -> SQLite | 并行完成顺序不确定，失败 cleanup 必须等待写入结束。 |
| Screening run -> chip results | latest 查询竞态可能把不同批次数据混合。 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Denial of Service | workflow fan-out | mitigate | 所有 query 通过 scheduled client；集成测试真实峰值。 |
| T-08-05 | Tampering | bootstrap cleanup | mitigate | allSettled 后统一 activate/cleanup，禁止迟到写入。 |
| T-08-09 | Tampering | chip provenance | mitigate | 按精确 screening run id 读取候选。 |
| T-08-02 | Information Disclosure | row errors | mitigate | 保留现有 SafeTushareError 和 sanitize 流程。 |
</threat_model>
