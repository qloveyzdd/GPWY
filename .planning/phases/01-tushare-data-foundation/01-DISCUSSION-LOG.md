## 个人访问保护

### Protection Method

| Option | Description | Selected |
|--------|-------------|----------|
| 应用访问密码 | 页面访问时输入一个简单密码，密码放在服务端环境变量里。实现简单，适合个人云端网页。 | Yes |
| 反向代理/服务器限制 | 用 Nginx Basic Auth、IP 白名单或防火墙限制访问。应用本身不做登录，部署配置更重要。 | |
| 先不做应用保护，只绑定内网/本机 | 适合只通过 SSH 隧道或内网访问；如果暴露公网就不够安全。 | |
| 由你决定 | 后续规划时按“个人使用、低复杂度、token 不暴露”的原则选择。 | |

**User's choice:** 应用访问密码，并使用 `.env` 设置密码。
**Notes:** 密码不得提交或暴露到前端。

### Protection Scope

| Option | Description | Selected |
|--------|-------------|----------|
| 保护整个网页 | 未输入密码时不能看页面，也不能触发刷新或查看结果。 | Yes |
| 只保护刷新接口 | 页面和结果可看，但刷新需要密码。不适合公网个人工具。 | |
| 保护刷新和结果，状态页可公开 | 可以公开看到服务是否运行，但数据和操作受保护。 | |
| 由你决定 | 默认按最小安全原则保护整个网页。 | |

**User's choice:** 保护整个网页。
**Notes:** 未通过访问密码前，不能查看页面、结果，也不能触发刷新。

### Failure Behavior

| Option | Description | Selected |
|--------|-------------|----------|
| 停留在登录页并显示通用错误 | 例如“密码错误”，不透露系统内部状态。 | Yes |
| 返回 401 页面 | 更像 API/Basic Auth 行为，体验较硬。 | |
| 由你决定 | 默认用通用错误，不泄露细节。 | |

**User's choice:** 停留在登录页并显示通用错误。
**Notes:** 错误信息要通用化，避免泄露内部状态。

## 数据源状态页

### Status Panel Scope

| Option | Description | Selected |
|--------|-------------|----------|
| 完整状态面板 | 显示 token 配置状态、Tushare 连接结果、股票基础信息样例、行情价格口径、筹码候选接口验证结果。 | Yes |
| 极简状态 | 只显示 Tushare 连接成功/失败和筹码接口可用/不可用。页面简单，但排查信息少。 | |
| 开发者详情优先 | 显示每个接口的请求时间、返回字段、样例数据和错误摘要。更适合调试，但首版页面会更工程化。 | |
| 由你决定 | 默认做完整状态面板，错误脱敏。 | |

**User's choice:** 完整状态面板。
**Notes:** 需要覆盖 token、连接、股票基础信息样例、行情口径和筹码候选接口验证。

### Sample Data Display

| Option | Description | Selected |
|--------|-------------|----------|
| 只展示脱敏摘要 | 例如接口名称、成功/失败、字段列表、样例股票代码/名称、错误类别。不展示原始完整响应。 | Yes |
| 展示完整原始响应 | 调试方便，但容易把不必要的数据和潜在敏感信息暴露到页面。 | |
| 页面摘要 + 服务端日志完整响应 | 页面干净，但日志需要注意不要记录 token。 | |
| 由你决定 | 默认页面只展示脱敏摘要。 | |

**User's choice:** 只展示脱敏摘要。
**Notes:** 不展示完整原始响应。

### Validation Refresh

| Option | Description | Selected |
|--------|-------------|----------|
| 页面按钮手动验证 | 用户点击“验证数据源”才调用 Tushare。符合首版手动触发原则，省额度。 | |
| 打开页面自动验证一次 | 体验直接，但每次访问都可能消耗 Tushare 额度。 | |
| 读取上次验证结果，按钮可重新验证 | 首屏不调用 Tushare，同时保留最近状态。稍微多一点缓存逻辑。 | Yes |
| 由你决定 | 默认按钮手动验证。 | |

**User's choice:** 读取上次验证结果，按钮可重新验证。
**Notes:** 首屏不调用 Tushare；重新验证由用户主动触发。

## 行情价格口径

### Initial Price Basis

| Option | Description | Selected |
|--------|-------------|----------|
| 前复权 | 更适合连续技术分析，分红送转后 MA 和高点不容易被除权缺口污染。 | |
| 未复权 | 与交易所原始行情一致，但遇到除权除息时技术指标可能失真。 | |
| 后复权 | 历史价格按当前价格体系调整，适合长期收益分析，但直觉上不如前复权常用于看图。 | |
| Phase 1 先实测再锁定 | 先验证 Tushare 能稳定提供哪些口径，再选择。更稳，但会把决策推给实现阶段。 | Yes |

**User's choice:** Phase 1 先实测再锁定。
**Notes:** 需要验证 Tushare 可稳定提供哪些价格口径。

### Selection Principle

| Option | Description | Selected |
|--------|-------------|----------|
| 能稳定获取前复权就用前复权，否则退回未复权 | 技术分析优先连续性，同时保留可用兜底。 | Yes |
| 优先未复权，只记录除权风险 | 数据最原始，但技术分析连续性较差。 | |
| 必须前复权，不可用就阻塞 | 口径最严格，但可能卡住 Phase 1。 | |
| 由你决定 | 默认按“能稳定获取前复权就用前复权，否则退回未复权”。 | |

**User's choice:** 能稳定获取前复权就用前复权，否则退回未复权。
**Notes:** 前复权优先，但不可稳定获取时不阻塞 Phase 1。

### UI Disclosure

| Option | Description | Selected |
|--------|-------------|----------|
| 明确显示当前口径和原因 | 例如“当前使用：前复权；原因：适合连续技术分析”。如果退回未复权，也显示原因和风险。 | Yes |
| 只在后台记录，不在页面展示 | 页面更简洁，但用户容易忘记指标口径。 | |
| 只在错误/警告时展示 | 当前复权成功时不显示，退回未复权时显示警告。 | |
| 由你决定 | 默认在状态页明确显示当前口径和原因。 | |

**User's choice:** 明确显示当前口径和原因。
**Notes:** 退回未复权时也要显示原因和风险。
