<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Next.js middleware | Untrusted visitors attempt to access protected page/API routes. |
| Browser -> login action | User-supplied password enters server validation. |
| Server config -> UI/API response | Secrets must be reduced to safe booleans and categories. |
| App -> npm registry | Package install supply chain crosses external registry boundary. |
| Server -> SQLite | Validation snapshot persistence crosses storage boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | `src/lib/config.ts` | mitigate | Keep `APP_PASSWORD` and `TUSHARE_TOKEN` in server-only module; tests assert token values do not render or appear in API JSON. |
| T-01-02 | Elevation | `src/middleware.ts` | mitigate | Validate HTTP-only signed session before allowing page or validation API access. |
| T-01-03 | Tampering | `src/lib/auth/session.ts` | mitigate | Sign session cookies with Node crypto HMAC and expiry. |
| T-01-04 | Information Disclosure | `src/components/status/status-workspace.tsx` | mitigate | Render only safe snapshot fields; no raw payloads, tokens, headers, or stack traces. |
| T-01-SC | Tampering | npm installs | mitigate | Execute package legitimacy checkpoint before install; stop if packages cannot be verified. |

</threat_model>
