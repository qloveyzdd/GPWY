# 风险研究

**领域：** 增量行情、受控并发和筹码分布
**研究日期：** 2026-06-25
**置信度：** 高

## 关键风险

### 1. 将并行误写成无限并发

**问题：** 对候选股票直接执行 `Promise.all`，同时启动数百个 Python 进程或请求。

**后果：** 内存和进程数激增、接口限频、失败率上升，整体反而更慢。

**预防：** 所有 provider 请求必须经过同一个有界调度器；初始并发使用保守配置，并记录队列长度、请求耗时和限频次数。

**验证：** 测试峰值在途请求数永远不超过配置值。

### 2. 增量存储继续保存已复权价格

**问题：** 前复权价格依赖最新交易日的复权因子，后续除权会改变历史价格基准。

**后果：** 新增一日后，旧日价格与新日价格不在同一基准，均线和筛选结果错误。

**预防：** 保存原始 OHLCV 和独立复权因子，读取时统一换算。

**验证：** 添加新的复权因子后，历史窗口价格和均线按新基准一致重算。

### 3. 把旧快照当作原始行情迁移

**问题：** 现有 `daily_bars` 没有对应复权因子，无法反推原始价格。

**后果：** v2 数据仓库从第一天就混入不同口径价格。

**预防：** 首次 v2 使用 provider 重新引导最近 60 个交易日；旧表只作为回滚来源，不导入新表。

### 4. 双日筹码分布产生陈旧价格档位

**问题：** 对同一日期仅 UPSERT 新档位，不删除响应中已不存在的旧档位。

**后果：** 图表混合新旧分布，总占比异常。

**预防：** 完整响应校验通过后，在一个事务内替换该股票该日期的全部档位。

### 5. “前一天”按自然日计算

**问题：** 周一、节假日或停牌股票没有前一自然日数据。

**后果：** 第二张图频繁空白或请求错误日期。

**预防：** 以前一条有效日线记录为准；交易日历只用于市场日期规划。

### 6. 筹码阶段失败污染行情刷新状态

**问题：** 将行情、筛选和筹码视为一个不可分割事务。

**后果：** 已成功的行情与筛选因少量筹码失败被判定为整体失败，重试时重复下载行情。

**预防：** 分阶段持久化状态；行情和筛选成功后即可发布，筹码保留独立的完成/部分/失败状态。

### 7. Python worker 无恢复边界

**问题：** worker 崩溃后无限重启，或在途请求永久悬挂。

**后果：** 任务看似运行但没有进展。

**预防：** 每项有超时、worker 有重启预算、在途请求可重新入队、总任务可终止。

### 8. 长网络请求持有 SQLite 事务

**问题：** 请求期间打开写事务。

**后果：** 页面读取或其他阶段写入被长时间锁住。

**预防：** 网络请求与数据库事务严格分离；只在完整响应准备好后执行短事务。

## 性能陷阱

| 陷阱 | 表现 | 预防 |
|------|------|------|
| 每请求启动 Python | CPU 不高但固定延迟明显、进程频繁出现 | 持久 worker 池 |
| 重复写完整 60 日快照 | 数据库快速膨胀，写入和备份越来越慢 | 唯一键增量表 |
| 对缓存命中仍发请求 | 第二次刷新和第一次一样慢 | 以股票/日期完整性判断跳过 |
| 为两天筹码发两次请求 | 请求量翻倍 | 使用 `start_date`/`end_date` 一次获取两日 |
| 严格串行重试 | 一个慢请求拖住后续全部任务 | 有界并发、单项超时、独立重试 |

## “看起来完成但实际未完成”

- [ ] 普通刷新在无新交易日时没有重复下载 60 日数据。
- [ ] 并发上限同时约束 REST 和 tinyshare。
- [ ] 限频错误触发退避，不立即形成重试风暴。
- [ ] 重启应用后能够识别并恢复未完成项目。
- [ ] 某一天筹码失败时，另一天图表仍可展示。
- [ ] 两个筹码图标明准确交易日期，不使用“昨天”歧义文案。
- [ ] 完整分布档位总数不再受旧的前三名约束。
- [ ] 全量重建不会覆盖最后一份可用数据，直到新缓存验证成功。
- [ ] 进度停滞、重试和失败都能从页面识别。

## 路线图映射

| 风险 | 应处理阶段 | 验证 |
|------|------------|------|
| 复权口径和旧数据迁移 | 数据模型阶段 | 除权场景单元测试、双读一致性 |
| 无限并发和限频 | 调度器阶段 | 虚拟时钟并发/退避测试 |
| worker 崩溃和悬挂 | provider 阶段 | 进程退出、超时、重启预算测试 |
| 重复下载和不可恢复 | 增量刷新阶段 | 两次连续刷新请求计数测试 |
| 陈旧筹码档位 | 筹码存储阶段 | 原子替换测试 |
| 双图部分失败 | UI 阶段 | 组件和浏览器测试 |
